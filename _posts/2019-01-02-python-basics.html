  • Python 3 was released in 2008.
  • Python is an interpreted language (in contrast to compiled languages like C,C++,Java) which means the interpreter will check the code line by line. No need of separate compilation and execution steps.
  • Python uses indentation to identify the blocks of code . Indentation in python is a Tab space/4 spaces (in contrast to other languages like C,C++,Java which uses { } to specify blocks of code ).
  • Python statements will not use any character to specify end of statement (in contrast to other languages like C,C++,Java which uses ; ). Statement in single line is considered as single line of code.
  • In python we need not declare type of variable. Python automatically identifies the type of variable based on the value assigned to it.
  • Python has 4 builtin data structures . List, Tuple, Set, Dictionary.
  • Python language is case sensitive i.e python , Python, PYTHON are considered as different words.

Python Architecture

Python architecture and its usual flow.
There are four steps that Python takes when we run python code.

  1. Lexing
  2. Parsing
  3. Compiling
  4. Interpreting

Lexing: Lexing is breaking the line of code we typed into tokens.Tokens are constants which represent the numeric values
E.g: PLUS, MINUS,NAME, NUMBER, EQUAL, LESS, DOT, GREATER, PERCENT

Parsing: The parser takes those tokens and generates a structure that shows their relationship to each other. For Python and many other langauges, this is the Abstract Syntax Tree (or AST).Think of your code as a tree. The top level is the root, a function might be a branch, a class is a branch and the class methods branch off that. The statements are leaves within a branch.

Compiling: The compiler takes the AST and turns it into one (or more) code objects(ByteCode). Bytecode is the result of compiling source code written in a language. Bytecode is the intermediary language between machine code (01010101) and the syntax tree. The Byte-code sequence is the thing cached in .pyc files.

Interpreting: It takes the byte-code sequence and converts it into machine-specific operations. This is why Python.exe and Python for mac and Linux are all seperate binaries. Some byte codes need OS specific handling and checks. Interpreter executes the code line by line in a REPL (Read-Evaluate-Print-Loop) fashion.

Python Flavours

  • CPython: Python implementation written in C, and is an interpreter.
  • Jython: Python implementation written in Java. A Jython program can import any Java class. It compiles to Java bytecode.
  • IronPython: Python implementation written in C#. It can function as an extensibility layer to application frameworks written in a .NET language.
  • Brython: Brython stands for Browser Python. It is an implementation of Python that runs in the browser.
  • RubyPython: Acts as a bridge between the Python and Ruby interpreters.
  • PyPy: Python implemented in Python.
  • MicroPython: Implementation of Python meant to run on a microcontroller. It uses a MicroPython board.

Various Python File Extensions

  • .py: Usual extension of python file.
  • .pyc: Compiled bytecode. This code is then executed by Python's virtual machine.
  • .pyd: Windows DLL(dynamic link library) file
  • .pyo: File created with optimizations
  • .pyw: Python script for Windows
  • .pyz: Python script archive.Packages all source code and dependencies into a single Python zip application.
